使用矩阵机器人程序编写练习调试
发表于 2019-12-02| 更新于 2019-12-05|黑科技
阅读量:

一．调试分析

第一次调试：选择第一张图示地图进行验证，结果输出9，答案为10，将输出n改为输出n+1；

第二次调试：选择第二张图示地图进行验证，目的在于验证循环步数是否正确，结果输出：机器人走了5步，走出了地图。在主程序while循环处设置断点，发现程序在进行到x,y=1时误判为边界，置robot[1][1]为3，边界判定条件出错。
修正：将判定继续行进的条件改为y - 1 >= 0 && y - 1 <= lie格式。

第三次调试：选择第二张图示地图进行验证，目的在于验证是否返回2，在return 3前添加printf语句，结果并未输出，发现原因是if语句顺序错误，导致无法执行第二条if（第一条if的判定范围包含第二条）
修正：交换if语句。

第四次调试：选择第二张图示地图进行验证，结果输出2，应该输出8
修正：输出改为beforexunhuan+1.

至此，程序完成

二．总结。

断点调试有时不需要每次都进入函数查看，重点围绕出错的步骤观察。
可以使用printf输出观察程序执行某个语句与否。
